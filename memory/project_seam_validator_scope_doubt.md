The fact: Justin questioned the value of on-pattern seam matching. His framing — "some drafting procedures probably have it built in to keep matching seams the same length. it was going to be a convenience for free-handing if that's even a thing in pattern making."

**Why:** Drafting from a published procedure (Aldrich, TextileBlog, Cornelius Quiring) typically lays out paired seams to be the same length by construction. The validator's value scales with how often Justin actually free-hands without a procedure — which may be approximately never for the v1 self-use case (Phase 6 is "redraft my trouser using the TextileBlog method"). If Justin is always following a procedure, the validator is fixing a problem he never has.

**How to apply:** When chunks 3.8 (seam edges) and 3.9 (seam-pair validator) come up for their feedback cycle:
- Don't assume the validator is locked in. Surface the question explicitly: does Justin actually want this, or is the data-model work for seam pairs sufficient on its own?

- Darts and dart-leg-extension-to-SA are unrelated to seam pairing. Treat them as fully decoupled when scoping.
